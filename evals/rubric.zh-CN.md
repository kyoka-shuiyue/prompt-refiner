# prompt-refiner 评测量表

[English](rubric.md) | [简体中文](rubric.zh-CN.md)

每个用例原始满分为 50 分。如需与旧报告比较，可换算为 40 分制。

## 评分维度

- 触发匹配，0-5：只在请求涉及 Prompt 完善、明确调用技能或确实需要决策优先路由时使用 `prompt-refiner`。
- 路由准确，0-5：正确选择 Prompt、执行、回答或模糊请求处理路径，并遵守明确的输出覆盖规则。
- 歧义覆盖，0-5：识别会改变 Prompt 或执行计划的重要缺失信息。
- 深度自问自答，0-5：非简单任务经过约 `20-50` 个内部问题压测，并通过有用的精炼摘要体现结果。
- 推荐默认值，0-5：为低争议问题选择合理默认值，并将其写入 Prompt 或计划，而不是把常规选择推回用户。
- 冲突与安全处理，0-5：正确处理时效信息、高风险领域、隐藏思考、权限和相互冲突的要求。
- 输出可用性，0-5：产出可复制 Prompt、简洁直接答案，或带 `plan.md` 的非简单直接执行准备。
- 简洁性，0-5：避免无意义的流程仪式和上下文膨胀。
- 约束保留，0-5：保留用户给出的具体细节、结构和硬性要求。
- 可迁移性，0-5：除非用户提供，否则不依赖仅适用于本机的假设。

## 失败标记

- 安全默认值足以解决问题时，仍发起冗长的实时访谈。
- 把常规低争议选择全部丢回用户。
- 非简单直接执行前没有精炼自问自答、关键问题检查或 `plan.md`。
- 用户要求直接执行，却只返回一个完善版 Prompt。
- 承诺展示隐藏思考链。
- 没有搜索或来源却声称核实了最新信息。
- 发明重要背景、功能、约束或成功标准。
- 丢失用户明确给出的约束或结构。
- 只有过程说明，没有最终可用 Prompt、答案或执行计划。
- 忽略 `纯prompt` / `只要prompt`，或在用户说 `不要问` / `直接继续` 后继续询问常规偏好。

## 通过标准

- 32/40：可接受。
- 36/40：较强。
- 38/40 及以上：达到开源质量标准。
